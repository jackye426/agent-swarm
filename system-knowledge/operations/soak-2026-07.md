## Follow-ups surfaced by the T-011/T-012 chain

1. **Planning can contradict the intake agreement** — contract draft/review must treat `requirements_summary` as binding constraints, and the verifier should receive it so contract-vs-requirements conflicts classify as `contract` on the first pass (T-012 burned 3 rework attempts classified as `implementation`).
2. **Chained tasks build on stale bases** — dependency branches are never merged (`GITHUB_CREATE_PR=false`), so T-012 could not see T-011's Express backend and re-added it. Need branch handoff: base dependent worktrees on the dependency's branch, or merge on COMPLETE.
3. **`.env` has a UTF-8 BOM on line 1** — `grep '^SUPABASE_URL='` silently fails; strip the BOM.

## Rules during soak

- Do **not** run `reseed:hygiene`, manual enqueues, or `pm2 restart` unless responding to an alert — the point is proving hands-off operation.
- Every alert received (Telegram or healthchecks.io) gets a row in the log above with time-to-alert.
- If something breaks silently (noticed by a human, not an alert), that is a **soak failure** — record it, fix detection, restart the window.
